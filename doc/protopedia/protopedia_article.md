## システム構成【画像アップ + Markdown解説】

- **画像**: `doc/protopedia/architecture_gcp.png`（1280x880・生成済み）をそのままアップロード（ハッカソンルールで構成図は必須）
- **解説文**: 以下を貼り付け

```markdown
**すべて Google Cloud・APIキーレス（ADC認証）構成**

- **Cloud Run**: 静的UI（Vanilla JS）と5本のAPI（extractTerms / explainTerm / generateNotes / generateMinutes / transcribeAudio）を1コンテナに同居。min-instances=0 でコストゼロ運用
- **Dictionary Dispatcher**: 固定辞書8分野＋プロジェクト辞書＋自分辞書。用語ごとに「辞書で即答」「自分辞書から再表示」「文脈AIに回す」を判断し、**どこでAIを使わないか**を決める
- **Vertex AI (Gemini 2.5 Flash)**: 用語の文脈込み説明・Full抽出・まとめ生成。Cloud Run のサービスアカウントの ADC で認証し、APIキーを一切持たない
- **Cloud Speech-to-Text**: 音声入力Beta。ブラウザで数秒ごとに録音した音声を Cloud Run 経由でキーレス中継
- **Cloud Build (CI/CD)**: main への push で自動ビルド → 新リビジョンを `--no-traffic` でカナリアデプロイ → candidate URL への本番スモークテスト（UI・セキュリティ応答・Vertex到達・STT中継）に合格したときだけトラフィック切替。「AIだけが静かに死ぬ」回帰（開発中に実際に発生）をゲートで検知する
- **エージェント実行ログ**: 辞書/AIの振り分け判断をUIのフロートパネルにリアルタイム表示。エージェントの自律判断を審査員がその場で確認できる
- 保存はブラウザの localStorage / sessionStorage（自分辞書・学習ワード）。サーバ側に個人データを持たない
```

以下を貼り付け:

```markdown
## ① 解決したい課題と背景

会議に出ると、知らない言葉が飛んでくる。知らない略語、知らない社内用語。しかも全員知ってる顔で進んでいく。

「それってどういう意味ですか？」と聞けばいい。正論。でも人間はそんなに強くない。毎回聞くと話の腰を折るし、自分だけ理解していない感じも出る。

だから作った。**その場では知ったかで乗り切り、あとでちゃんと追いつくためのAIエージェント**、それが「知ったかくん」です。議事録ツールではなく、「この会議で自分は何を知らなかったのか」を見える化します。

## ② 想定する利用ユーザー

- 転職・異動・配属直後で、会議の専門用語・社内用語の波についていけない人
- 複数分野のプロジェクトを兼務していて、会議ごとに用語の世界が切り替わる人
- 「わからなかったら聞いてね」と言いつつ、実際は聞きにくい空気が流れがちなチーム

## ③ プロダクトの特徴

- 会議ログをリアルタイムに流し、発言から専門用語っぽいものを抽出してチップ化
- IT・医療・福祉・経営・製造・服飾・ギャル・オタク系の分野別辞書に振り分け（Dictionary Dispatcher）
- 固定辞書にある用語は即答、ない用語は会議の文脈込みでGeminiが説明
- 「知らない」「気になる」のクリックを学習シグナルとして記録
- 会議後にクリックした語だけを集めた「知ったかまとめ」を生成（議事録とは別物）
- 学習ワードを「自分辞書」に登録し、次回の会議で優先的に再表示
- 音声入力Beta（Web Speech API / Google Cloud Speech-to-Text）、議事録生成もあり
- Google Meet 取込のMock導線（本番連携ルートは設計済み）

### 工夫したところ：AIプロダクトなのに、AIをなるべく呼ばない

会議中のツールは「賢さ」より「止まらなさ」が命。AIの応答を30秒待っている間に会議は次の話題に行ってしまう。

そこで抽出を2段階に分けた。

1. **Fast抽出**: 辞書とルールベースのディスパッチャで即座にチップを出す（AIを待たない）
2. **Full抽出**: 一定間隔でAI込みの抽出を走らせ、取りこぼしを補う

さらに Dictionary Dispatcher が用語ごとに「固定辞書で即答」「自分辞書から再表示」「文脈AIに回す」を判断する。**どこでAIを使わないかを決める**のが、このエージェントの一番エージェントらしいところ。この判断過程は画面上の「エージェント実行ログ」パネルにリアルタイムで流れるので、動いているところをそのまま見てもらえます。

AIが落ちても辞書とヒューリスティックでUIは止まらない。会議ログはプロンプトインジェクション対策として命令ではなくデータとして扱い、入力サイズ制限・Rate Limit・CORS制御も入れた。

### まわす：一度踏んだ地雷は二度踏まない

デプロイは Cloud Build のカナリアパイプライン。新リビジョンはまずトラフィック0%で立ち上がり、本番相当のスモークテスト（UI表示・画像配信・セキュリティ応答・Vertex Gemini到達・音声認識中継）に合格したときだけ本番に昇格する。スモークの各チェックは開発中に実際に踏んだ事故（AIだけが静かに死ぬ環境変数の欠落、画像がアップロードから漏れる設定ミス）に対応している。事故は防げないが、同じ事故を二度踏むのは選択なので。

### 自分辞書 — 評価ではなく成長ログ

「知らない」を押した語は自分辞書に育っていく。これは弱点リストではなく成長ログ。昨日つまずいた言葉を、今日ちょっと拾いやすくする。「この人は理解力が低い」みたいな判定は作らない。人間を管理しようとすると、だいたいろくなことにならない。
```
