## 作品名

知ったかくん (Meeting Whisperer)

## 概要（キャッチコピー・短文）

会議中の「知らない言葉」をリアルタイムに拾い、固定辞書・自分辞書・AI説明に振り分けて、その場は知ったかで乗り切り、あとでちゃんと追いつくための会議支援AIエージェント。議事録ツールではなく「この会議で自分は何を知らなかったのか」を見える化する。

## ストーリー（本文）

> ①=「なにこれ」、②=「想定ユーザー」、③=「できること」以降が対応。

### なにこれ（解決したい課題と背景）

会議に出ると、知らない言葉が飛んでくる。知らない略語、知らない社内用語。しかも全員知ってる顔で進んでいく。

「それってどういう意味ですか？」と聞けばいい。正論。でも人間はそんなに強くない。毎回聞くと話の腰を折るし、自分だけ理解していない感じも出る。

だから作った。**その場では知ったかで乗り切り、あとでちゃんと追いつくためのAIエージェント**、それが「知ったかくん」です。

### 想定ユーザー

- 転職・異動・配属直後で、会議の専門用語・社内用語の波についていけない人
- 複数分野のプロジェクトを兼務していて、会議ごとに用語の世界が切り替わる人
- 「わからなかったら聞いてね」と言いつつ、実際は聞きにくい空気が流れがちなチーム

### できること（プロダクトの特徴）

- 会議ログをリアルタイム風に流し、発言から専門用語っぽいものを抽出してチップ化
- IT・医療・福祉・経営・製造・服飾・ギャル・オタク系の分野別辞書に振り分け（Dictionary Dispatcher）
- 固定辞書にある用語は即答、ない用語は会議の文脈込みでGeminiが説明
- 「知らない」「気になる」のクリックを学習シグナルとして記録
- 会議後にクリックした語だけを集めた「知ったかまとめ」を生成（議事録とは別物）
- 学習ワードを「自分辞書」に登録し、次回の会議で優先的に再表示
- 音声入力Beta（Web Speech API / Google Cloud Speech-to-Text）/ 議事録生成もあり
- Google Meet 取込のMock導線（本番連携ルートは設計済み）

### 工夫したところ：AIプロダクトなのに、AIをなるべく呼ばない

会議中のツールは「賢さ」より「止まらなさ」が命。AIの応答を30秒待っている間に会議は次の話題に行ってしまう。

そこで抽出を2段階に分けた。

1. **Fast抽出**: 辞書とルールベースのディスパッチャで即座にチップを出す（AIを待たない）
2. **Full抽出**: 一定間隔でAI込みの抽出を走らせ、取りこぼしを補う

さらに Dictionary Dispatcher が用語ごとに「固定辞書で即答」「自分辞書から再表示」「文脈AIに回す」を判断する。**どこでAIを使わないかを決める**のが、このエージェントの一番エージェントらしいところ。

AIが落ちても辞書とヒューリスティックでUIは止まらない。会議ログはプロンプトインジェクション対策として命令ではなくデータとして扱い、入力サイズ制限・Rate Limit・CORS制御も入れた。

### システム構成

```
[ブラウザ (Vanilla JS)]
   │  演出デモ / 音声入力Beta / Meet Mock
   ▼
[Cloud Run (Node.js / TypeScript)]
   ├─ 静的UI配信
   ├─ /api/extractTerms    … 用語抽出（Fast: 辞書 / Full: Gemini）
   ├─ /api/explainTerm     … 文脈込み用語説明
   ├─ /api/generateNotes   … 知ったかまとめ生成
   ├─ /api/generateMinutes … 議事録生成
   └─ /api/transcribeAudio … 音声文字起こし中継
   │
   ├─ Dictionary Dispatcher（固定辞書8分野 + プロジェクト辞書 + 自分辞書）
   ├─▶ [Vertex AI — Gemini 2.5 Flash]（ADC認証・APIキーレス）
   └─▶ [Cloud Speech-to-Text]（ADC認証・APIキーレス）

保存: localStorage / sessionStorage（自分辞書・学習ワード）
```

- **Cloud Run**: 静的UIと5本のAPIを1コンテナに同居。min-instances=0 でコストゼロ運用
- **Vertex AI (Gemini 2.5 Flash)**: APIキーを一切使わず、Cloud RunのサービスアカウントのADCで認証
- **Cloud Speech-to-Text**: 音声入力Beta。ブラウザで数秒ごとに録音した音声をCloud Run経由で中継（キーレス）
- **Cloud Build (CI/CD)**: main への push で自動ビルド → 新リビジョンを `--no-traffic` でカナリアデプロイ → 本番スモークテストに合格したときだけトラフィック切替。AIが静かに死ぬ回帰（過去に実際に発生）をゲートで検知する
- **エージェント実行ログ**: 辞書/AIの振り分け判断（どこでAIを呼ばなかったか）をUIパネルにリアルタイム表示
- 公開デモは「AI可視化モード」で稼働中: 用語クリック時の説明は辞書ヒットでも常にGeminiが文脈込みで生成（説明カードの source 表示が `vertex_gemini` になるので、AIが動いていることをその場で確認できます。AI失敗時は辞書に自動フォールバック）

### 自分辞書 — 評価ではなく成長ログ

「知らない」を押した語は自分辞書に育っていく。これは弱点リストではなく成長ログ。昨日つまずいた言葉を、今日ちょっと拾いやすくする。「この人は理解力が低い」みたいな判定は作らない。人間を管理しようとすると、だいたいろくなことにならない。
