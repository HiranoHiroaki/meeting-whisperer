# Decision Log

## 2026-05-07 MVP Fixed Decisions

### 1. 用語抽出精度方針
- MVPは「怪しいものを広めに拾う」を採用する。
- 目的は完璧な辞書化ではなく、会議を止めない補助である。
- 抽出スコアは以下を基準にする。
  - 英大文字連続
  - CamelCase
  - 社内略語らしい短語
  - 会議ログ内の頻出未知語
- すべて「知らない候補」として提示し、断定しない。

### 2. ハルシネーション対策
- 社内語・固有語は推定扱いにする。
- 断定文は禁止する。
- 説明テンプレート: 「この会議では〜を指している可能性があります。」

### 3. クリック意味の分離
- クリック種別を分ける。
  - `unknown`: 本当に知らない
  - `interest`: 気になる / もっと知りたい
- UIは2アクションを明示する。

### 4. レイテンシ方針
- `extractTerms` は先出しする。
- 詳細説明はクリック時に遅延生成する。
- 初回で全説明を生成しない。

### 5. 候補表示ルール
- 候補語の表示は最大5件固定。
- 並び順はスコア順。
- スコアは新規性・特殊性・略語度を重視する。

### 6. 差別化メッセージ
- 会議要約AIとの比較で「個人化」を前面に出す。
- コアメッセージ: 「あなたが分かっていなかった部分だけを補完する」。

### 7. MVP禁止事項
- Teams SDK / Bot 連携
- 音声リアルタイム連携
- WebSocket中心のリアルタイム基盤

### 8. MVP完了条件
以下が動作すれば提出可能とする。
1. 会議ログ入力
2. 用語抽出
3. 詳細生成
4. クリック記録
5. 今日の知ったかまとめ
6. `knowledge_profile.md` 生成

### 9. Knowledge Profile ルール
- `unknown` が同一語で3回: `Weak` へ
- `interest` が同一語で1回: `Curious` へ
- `Recently Learned` は7日間非接触で自動除外

### 10. プロダクト姿勢
- `knowledge_profile` は評価や監視目的に使わない。
- 「理解力が低い」のような判定は行わない。
- 成長ログとして扱う。

### 11. データ方針 (MVP)
- 保存はローカル優先 (localStorage / ローカルファイル)
- ユーザーがいつでも履歴とプロフィールを削除できること
